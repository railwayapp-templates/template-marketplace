# Deploy WAHA + n8n (w/ persistent nodes) on Railway

Automate your WhatsApp with workflows

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/template/waha-n8n)

## About

**WAHA + n8n** lets you automate WhatsApp using powerful, visual workflows. Connect WhatsApp events to APIs, databases, CRMs, AI tools, and internal services without writing glue code. This template ships a fully integrated setup, ready to use in minutes.

Hosting WAHA alongside n8n gives you a complete WhatsApp automation backend running in the cloud. WAHA handles WhatsApp session management and messaging, while n8n orchestrates workflows, logic, and integrations. This template automates the setup, installs the WAHA n8n node automatically, and exposes both services with minimal configuration. Ideal for production, CI environments, and rapid prototyping, with optional persistence via PostgreSQL.

## What gets deployed

| Service | Source | Type |
|---------|--------|------|
| n8nio/n8n | `n8nio/n8n` | Web service |
| Postgres | `ghcr.io/railwayapp-templates/postgres-ssl:17` | Database |
| devlikeapro/waha | `devlikeapro/waha` | Web service |

## Environment variables

| Variable | Service | Default | Description |
| --------- | ------- | ------- | ----------- |
| `PORT` | n8nio/n8n | 5678 | Internal port used by n8n |
| `DB_TYPE` | n8nio/n8n | postgresdb | Informs n8n to use PostgreSQL as the database backend |
| `WEBHOOK_URL` | n8nio/n8n | - | Public URL used to register and receive webhooks |
| `NODE_OPTIONS` | n8nio/n8n | --max_old_space_size=8192 | Increases Node.js memory limit (in MB) |
| `N8N_TRUST_PROXY` | n8nio/n8n | true | Trusts Railway reverse proxy headers (required for HTTPS and correct URLs) |
| `DB_POSTGRESDB_HOST` | n8nio/n8n | - | Database host provided by Railway Postgres plugin |
| `DB_POSTGRESDB_USER` | n8nio/n8n | (secret) | Database user provided by Railway Postgres plugin |
| `N8N_ENCRYPTION_KEY` | n8nio/n8n | - | Encryption key for credentials; must remain stable across restarts |
| `N8N_BASIC_AUTH_USER` | n8nio/n8n | (secret) | Default admin username for Basic Auth |
| `N8N_EDITOR_BASE_URL` | n8nio/n8n | - | Public base URL for the n8n editor |
| `N8N_BASIC_AUTH_ACTIVE` | n8nio/n8n | true | Enables Basic Auth to protect the n8n editor and API |
| `DB_POSTGRESDB_DATABASE` | n8nio/n8n | - | Database name provided by Railway Postgres plugin |
| `DB_POSTGRESDB_PASSWORD` | n8nio/n8n | (secret) | Database password provided by Railway Postgres plugin |
| `N8N_BASIC_AUTH_PASSWORD` | n8nio/n8n | (secret) | Secure random admin password generated by Railway |
| `N8N_REINSTALL_MISSING_PACKAGES` | n8nio/n8n | true | Automatically reinstalls missing community nodes on startup |
| `N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS` | n8nio/n8n | true | Enforces secure permissions on n8n settings files |
| `POSTGRES_DB` | Postgres | railway | Default database created when image is started. |
| `DATABASE_URL` | Postgres | - | URL to connect to Postgres database. |

| `WAHA_LOG_LEVEL` | devlikeapro/waha | info | Log verbosity level (`info` by default). |

| `WAHA_DASHBOARD_ENABLED` | devlikeapro/waha | True | Enables the management dashboard (`True` by default). |
| `WHATSAPP_START_SESSION` | devlikeapro/waha | default | Name of the WhatsApp session that is automatically started on service boot. |
| `WAHA_DASHBOARD_PASSWORD` | devlikeapro/waha | (secret) | Dashboard password, automatically generated during deployment. |
| `WAHA_DASHBOARD_USERNAME` | devlikeapro/waha | (secret) | Username for the WAHA web dashboard (default: `admin`). |

| `WHATSAPP_SWAGGER_PASSWORD` | devlikeapro/waha | (secret) | Password to access the Swagger UI, automatically generated and unique per deployment. |
| `WHATSAPP_SWAGGER_USERNAME` | devlikeapro/waha | (secret) | Username to access the Swagger UI (default: `admin`). |
