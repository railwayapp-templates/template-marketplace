# Deploy Directus on Railway

Convert any SQL database into a powerful API and CMS platform

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/directus-official)

## About

Directus is an open-source headless CMS and data platform that transforms any SQL database into REST and GraphQL APIs with a visual interface. Design your data model visually, build powerful APIs automatically, and enable anyone to author content, manage media, and visualize data, all without vendor lock-in.

Hosting Directus on Railway provides a complete infrastructure stack with minimal configuration. The template automatically provisions a PostgreSQL database, Redis cache, and S3-compatible bucket storage, all connected via Railway's private network for secure, zero-egress communication. Directus runs on Node.js and connects to your database to provide a visual interface and automatic API generation. File uploads and media assets are stored in Railway's scalable S3-compatible bucket storage, ensuring files persist across deployments and work seamlessly with multiple instances. All services are pre-configured with best practices, including caching, connection pooling, S3 storage, and security settings. Railway handles scaling, monitoring, and deployment automatically, so you can focus on building your application rather than managing infrastructure.

## Environment variables

| Variable | Service | Default | Description |
| --------- | ------- | ------- | ----------- |

| `REDIS` | Directus | - | Redis connection for caching. Automatically configured when you add Redis service. |
| `SECRET` | Directus | (secret) | Encryption key for securing your data. Railway automatically generates this for you. |
| `DB_CLIENT` | Directus | pg | Database type (PostgreSQL). This is pre-configured. |
| `LOG_STYLE` | Directus | raw | Format logs for easy reading in Railway's dashboard. |
| `PUBLIC_URL` | Directus | - | Your public Directus URL. This is where you'll access the admin panel. Automatically set when HTTP proxy is enabled on the service. |
| `CACHE_STORE` | Directus | redis | Cache storage type. Pre-configured for Redis. |
| `CACHE_ENABLED` | Directus | true | Enable caching to speed up your Directus instance. |
| `STORAGE_S3_KEY` | Directus | - | Storage access key. Automatically provided by Railway. |
| `CACHE_AUTO_PURGE` | Directus | true | Automatically clear cache when content changes. |
| `STORAGE_LOCATIONS` | Directus | s3 | Use cloud storage for files. Set to 's3' for Railway's storage service. |
| `STORAGE_S3_BUCKET` | Directus | - | Storage bucket name. Automatically created by Railway. |
| `STORAGE_S3_DRIVER` | Directus | s3 | Storage driver type. Set to 's3' for S3-compatible storage. |
| `STORAGE_S3_REGION` | Directus | - | Storage region. Automatically configured by Railway. |

| `WEBSOCKETS_ENABLED` | Directus | true | Enable real-time features like live updates and notifications. |
| `EXTENSIONS_LOCATION` | Directus | s3 | Where Directus extensions are stored. Set to 's3' to store extensions in cloud storage, enabling them to work across multiple instances if you scale your deployment. |
| `STORAGE_S3_ENDPOINT` | Directus | - | Storage endpoint URL. Automatically configured by Railway. |
| `DB_CONNECTION_STRING` | Directus | - | Database connection. Automatically configured when you add PostgreSQL service. |

| `POSTGRES_INITDB_ARGS` | PostGIS | -c ssl=on -c ssl_cert_file=/etc/ssl/certs/ssl-cert-snakeoil.pem -c ssl_key_file=/etc/ssl/private/ssl-cert-snakeoil.key | PostgreSQL initialization arguments for SSL configuration |

## Configuration

- **Healthcheck:** `/server/health`
- **Networking:** Public domain with automatic HTTPS
- **Start command:** `/bin/sh -c "rm -rf $RAILWAY_VOLUME_MOUNT_PATH/lost+found/ && exec docker-entrypoint.sh redis-server --requirepass $REDIS_PASSWORD --save 60 1 --dir $RAILWAY_VOLUME_MOUNT_PATH"`
- **TCP Proxies:** 6379
- **Volume:** `/data`
- **TCP Proxies:** 5432
- **Volume:** `/var/lib/postgresql/data`
