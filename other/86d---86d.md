# Deploy 86d on Railway

Dynamic commerce infrastructure built from modular components.

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/86d)

## About

**86d Store** is the open-source 86d commerce app: a Next.js storefront plus admin, PostgreSQL + Prisma, and a modular plugin system (cart, checkout, catalog, and more). Single-tenant; optional **86d.app** API for remote config and SSO.

Run the **Docker image** with **PostgreSQL**. On start it applies **Prisma migrations**, optionally **seeds** demo data (`AUTO_SEED`), then serves the app on **`PORT`** (default 3000). Set **`DATABASE_URL`**, **`BETTER_AUTH_SECRET`**, and **`STORE_ID`**. Configure **`STORAGE_PROVIDER`** (`local`, `s3`, or `vercel`) for uploads. **`/api/health`** is for health checks.

## What gets deployed

| Service | Source | Type |
|---------|--------|------|
| Postgres | `ghcr.io/railwayapp-templates/postgres-ssl:18` | Database |
| 86d | [86d-app/86d](https://github.com/86d-app/86d) | Web service |

## Environment variables

| Variable | Service | Default | Description |
| --------- | ------- | ------- | ----------- |
| `POSTGRES_DB` | Postgres | railway | Default database created when image is started. |
| `DATABASE_URL` | Postgres | - | URL to connect to Postgres database. |
| `POSTGRES_USER` | Postgres | (secret) | User to connect to Postgres DB |
| `POSTGRES_PASSWORD` | Postgres | (secret) | Password to connect to DB |
| `DATABASE_PUBLIC_URL` | Postgres | - | Public URL to connect to Postgres database, used by the Data panel. |
| `APP_URL` | 86d | - | The public service or customer domain, of the form `example.up.railway.app` |
| `STORE_ID` | 86d | - | When set (valid UUID), store config is fetched from the 86d API instead of template/config.json |
| `AUTO_SEED` | 86d | - | Set to "true" to automatically seed the database with demo data |
| `86D_API_KEY` | 86d | (secret) | 86d API (optional — when STORE_ID is set, fetch store config from hosted API) |
| `86D_API_URL` | 86d | https://api.86d.app | 86d API (optional — when STORE_ID is set, fetch store config from hosted API) |
| `S3_SECRET_KEY` | 86d | (secret) | - |
| `RESEND_API_KEY` | 86d | (secret) | https://resend.com/api-keys |
| `BETTER_AUTH_URL` | 86d | - | The public service or customer domain, of the form `example.up.railway.app` |
| `BETTER_AUTH_SECRET` | 86d | (secret) | A secret value used for encryption and hashing. It must be at least 32 characters and generated with high entropy. |
