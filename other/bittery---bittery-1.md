# Deploy bittery on Railway

Zero-knowledge password manager

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/bittery-1)

## About

**Bittery** is a source-available, zero-knowledge password manager. Passwords, notes, cards, and TOTP secrets are encrypted on your device before they reach the server—so plaintext vault data never leaves client devices. It supports web, desktop, mobile, and browser extension clients, with optional self-hosting or Bittery Cloud.

Hosting Bittery means running a **Caddy reverse proxy**, a **Rust API server**, and a **web vault** (static SPA behind nginx), backed by **PostgreSQL**. Caddy terminates TLS and routes API traffic (`/rpc`, `/sync`, `/cdn`, etc.) to the server and everything else to the web app on a single domain. The server handles authentication (SRP), encrypted vault sync, teams, and share links; it does not decrypt user data. Database migrations run automatically on server startup. The Railway template pre-wires `DATABASE_URL` and object-storage bucket variables—after deploy you mainly set `DOMAIN` on the Caddy service and optionally customize `JWT_SECRET` or `CORS_ORIGIN`. **Redis/Valkey** is optional for cross-instance SSE sync and distributed rate limiting. Official images: `ghcr.io/bittery-org/bittery-server:latest`, `ghcr.io/bittery-org/bittery-web:latest`, and `caddy:2-alpine` with the repo's [`Caddyfile`](https://github.com/bittery-org/bittery/blob/main/deploy/docker/Caddyfile).

## What gets deployed

| Service | Source | Type |
|---------|--------|------|
| Server | `ghcr.io/bittery-org/bittery-server:latest` | Worker |
| Reverse-Proxy | [bittery-org/bittery](https://github.com/bittery-org/bittery) (root: /deploy/railway) | Web service |
| Postgres | `ghcr.io/railwayapp-templates/postgres-ssl:18` | Database |
| Web | `ghcr.io/bittery-org/bittery-web:latest` | Worker |

| Variable | Service | Default | Description |
| --------- | ------- | ------- | ----------- |
| `PORT` | Server | 3000 | The server port |
| `JWT_SECRET` | Server | (secret) | - |
| `CORS_ORIGIN` | Server | - | This configures your cors origins |
| `WEB_APP_URL` | Server | - | The url to your web app |
| `BITTERY_MODE` | Server | self-hosted | Leave it as self-hosted |
| `DATABASE_URL` | Server | - | Your database connection |
| `BITTERY_STORAGE_BUCKET` | Server | - | Enter your bucket name here |
| `BITTERY_STORAGE_REGION` | Server | - | Enter your bucket region here |
| `BITTERY_STORAGE_CDN_URL` | Server | - | Enter your CDN URL here |
| `BITTERY_STORAGE_ENDPOINT` | Server | - | Enter your bucket endpoint here |
| `BITTERY_CLOUD_PUBLIC_SIGNUP` | Server | false | Leave as is |
| `BITTERY_CLOUD_BILLING_ENABLED` | Server | false | Leave as is |
| `BITTERY_STORAGE_ACCESS_KEY_ID` | Server | - | Enter your bucket access key id here |
| `BITTERY_STORAGE_SECRET_ACCESS_KEY` | Server | (secret) | Enter your bucket secret access key here |
| `PORT` | Reverse-Proxy | 8080 | The reverse proxy port |
| `WEB_PORT` | Reverse-Proxy | - | The web app port |
| `SERVER_PORT` | Reverse-Proxy | - | The bittery server ports |
| `WEB_PRIVATE_DOMAIN` | Reverse-Proxy | - | Auto generated by railway |
| `SERVER_PRIVATE_DOMAIN` | Reverse-Proxy | - | Auto generated by railway |
| `POSTGRES_DB` | Postgres | railway | Default database created when image is started. |
| `DATABASE_URL` | Postgres | - | URL to connect to Postgres database. |
| `POSTGRES_USER` | Postgres | (secret) | User to connect to Postgres DB |
