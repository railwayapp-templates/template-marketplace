# Deploy Postbase (v0.01) on Railway

The open-source alternative to Firebase and Supabase

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/deploy/postbase-v001)

## About

Postbase is a self-hosted auth and database platform for Next.js. Drop it into your stack, configure 25+ auth providers from a dashboard, and connect your app with a single SDK call. Think self-hosted Supabase or Clerk — you own the data, you control the infra.

Postbase ships as a single Docker container bundling PostgreSQL 18 (with pg_cron and pgmq), MinIO object storage, and the Next.js dashboard — all managed by supervisord. Deploying on Railway means one service, one attached volume, and a handful of environment variables. Railway detects the root Dockerfile automatically. You attach a volume at `/data` to persist your database and object storage, set a few secrets, generate a public domain, and you're live. No separate database service or storage bucket required — everything runs together out of the box.

## Environment variables

| Variable | Default | Description |
| --------- | ------- | ----------- |
| `AUTH_SECRET` | (secret) | A base secret string used for cryptographic operations across the application. It acts as the foundational key for generating and verifying secure tokens. In production, this should be a long, random, and securely stored value to prevent unauthorised access or token forgery. |
| `POSTGRES_DB` | postbase | Specifies the name of the PostgreSQL database that your application will connect to. This is the logical container where all your data, tables, and schemas are stored. |
| `POSTGRES_USER` | (secret) | Defines the username used to authenticate with the PostgreSQL database. This user must have the appropriate permissions to read from and write to the specified database. |
| `NEXTAUTH_SECRET` | (secret) | A secret key used by authentication frameworks (such as NextAuth) to sign and encrypt session tokens, cookies, and authentication data. |
| `POSTGRES_PASSWORD` | (secret) | The password associated with the PostgreSQL user. It is used together with POSTGRES_USER to establish a secure connection to the database. In a production environment, this should never be a simple or default value. |
| `POSTBASE_JWT_SECRET` | (secret) | This secret is used to sign JSON Web Tokens (JWTs) generated by your backend (Postbase). JWTs are typically used for stateless authentication between your client and server. |
