## About

Deploy and Host Alquimia Stack on Railway
What is Alquimia Stack?

Alquimia is a Low-Code/No-Code platform for building and governing AI agents in enterprise environments. Users can design agents without writing code, deploy them on their own infrastructure, and maintain full control over data and models, ensuring auditability and regulatory compliance.

About Hosting Alquimia Stack

Alquimia Stack consists of Studio, Runtime, Registry, Observability, Governance, and SDK + CLI. To deploy it, you need a Kubernetes cluster that supports GPU scheduling, model serving, and events, along with identity, secrets management, and observability systems such as OpenTelemetry. Railway provides the infrastructure layer, managing configuration and enabling both vertical and horizontal scaling without manual intervention. By hosting it on Railway, your agent stack runs in your controlled environment, with full traceability and no third-party dependency.

Common Use Cases
Automating customer support with agents that integrate with CRM and internal systems.
Orchestrating workflows across corporate applications using events and delegations.
Building no-code AI prototypes and moving them to production without rewriting logic.
Dependencies for Hosting Alquimia Stack
Kubernetes, including GPU scheduling and model serving
OpenTelemetry, for traces and metrics
Keycloak, for SSO
Vault, for secrets management
Event bus and storage, for the runtime layer

Why Deploy Alquimia Stack on Railway?

Railway is a unified platform for deploying your complete stack. Railway hosts your infrastructure, removing the need for manual configuration and enabling both vertical and horizontal scaling. By deploying Alquimia Stack on Railway, you move closer to a full-stack solution with minimal operational overhead: servers, databases, AI agents, and more, all managed in one place.

## Configuration

- **Start command:** `celery -A src.celery_app worker --loglevel=info`
- **Start command:** `sh -lc 'vault server -dev -dev-listen-address=0.0.0.0:8200 -dev-root-token-id="$VAULT_DEV_ROOT_TOKEN_ID"'`
- **Healthcheck:** `/v1/sys/health`
- **Volume:** `/qdrant/storage`
- **Volume:** `/data/files`
- **Volume:** `/var/lib/postgresql/data`
- **Healthcheck:** `/`
- **Networking:** Public domain with automatic HTTPS
- **Volume:** `/app/.data`
- **Start command:** `/bin/sh -lc 'sleep 10; mc alias set minio http://minio.railway.internal:9000 "$MINIO_ROOT_USER" "$MINIO_ROOT_PASSWORD"; mc mb --ignore-existing minio/alquimia'`
- **Healthcheck:** `/health/liveness`
- **Volume:** `/data`
- **Volume:** `/var/lib/registry`
- **Start command:** `minio server --console-address :9001 /data`
